En este apéndice, se presenta una extensión del Capítulo \ref{c5} el cual consiste en un algoritmo para manejar las dependencias entre los requisitos funcionales y los requisitos no-funcionales de la aplicación Web en un contexto orientado a objetivos (\emph{goal-oriented}). Con el algoritmo, es posible comprender cuál es el impacto en los requisitos procedente de un cambio en la estructura de modelos conceptuales que conforman la aplicación Web, así como saber qué requisitos necesitan ser implementados para cumplir, en medida de lo posible, los propósitos establecidos en el análisis orientado a objetivos. En esta extensión se describe la implementación del editor gráfico WebREd para la especificación de requisitos así como la implementación del algoritmo para analizar el impacto en los requisitos derivado de un cambio en los modelos conceptuales de la aplicación Web.
